AWS Deployment

Overview

The FFJ Consulting LLC website is deployed on AWS using S3 for static hosting, CloudFront for CDN and HTTPS, and Route 53 for DNS management.

Current Production Setup

  • Domain: ffjconsultingllc.com
  • S3 Bucket: ffj-consulting-website
  • CloudFront Distribution: E3545N3N8YO2FZ
  • Region: us-east-1
  • SSL Certificate: AWS Certificate Manager (ACM)

AWS Architecture

Components

  • Amazon S3: Static website hosting
  • Amazon CloudFront: CDN and HTTPS termination
  • Route 53: DNS management
  • ACM: SSL/TLS certificates

Data Flow

User Request
    ↓
Route 53 (DNS)
    ↓
CloudFront (CDN + HTTPS)
    ↓
S3 Bucket (Static Files)
    ↓
User Response

Initial Setup

Prerequisites

  • AWS Account with appropriate permissions
  • AWS CLI installed and configured
  • Domain registered (or use Route 53)
  • AWS profile configured (e.g., my-sso)

S3 Bucket Setup

  1. Create Bucket: 
    aws s3 mb s3://ffj-consulting-website --region us-east-1 --profile my-sso
  2. Enable Static Website Hosting: 
    aws s3 website s3://ffj-consulting-website \
    --index-document index.html \
    --error-document index.html \
    --profile my-sso
  3. Configure Public Access: 
    aws s3api put-public-access-block \
    --bucket ffj-consulting-website \
    --public-access-block-configuration \
    "BlockPublicAcls=false,IgnorePublicAcls=false,BlockPublicPolicy=false,RestrictPublicBuckets=false" \
    --region us-east-1 \
    --profile my-sso
  4. Set Bucket Policy: 
    aws s3api put-bucket-policy \
    --bucket ffj-consulting-website \
    --policy file://bucket-policy.json \
    --profile my-sso

CloudFront Setup

Create Distribution

CloudFront provides CDN and HTTPS capabilities:

  • Origin: S3 website endpoint
  • SSL Certificate: From ACM
  • Custom Domain: ffjconsultingllc.com
  • Error Pages: Redirect 404 to index.html (for SPA routing)

CloudFront Configuration

  • Distribution ID: E3545N3N8YO2FZ
  • Domain: d1129gv8vb7t5w.cloudfront.net
  • Price Class: North America and Europe
  • Default Root Object: index.html

Domain & SSL Setup

Route 53 Configuration

  1. Register Domain: Via Route 53 or external registrar
  2. Create Hosted Zone: For DNS management
  3. Update Nameservers: Point domain to Route 53 nameservers

SSL Certificate

  1. Request Certificate: In ACM (us-east-1 region)
  2. Domain Validation: Add DNS validation records
  3. Wait for Validation: Usually 5-30 minutes
  4. Attach to CloudFront: Use certificate ARN in distribution

DNS Records

Create A record (alias) pointing to CloudFront:

  • Type: A (Alias)
  • Name: ffjconsultingllc.com
  • Alias Target: CloudFront distribution
  • Also create: www.ffjconsultingllc.com (CNAME or A record)

Deployment Process

Automated Deployment

Use the deployment script:

python3 deploy.py

Or:

./deploy-with-new-domain.sh

Manual Deployment Steps

  1. Build Frontend: 
    cd frontend
npm run build
  2. Upload to S3: 
    aws s3 sync frontend/dist/ s3://ffj-consulting-website \
    --delete \
    --profile my-sso \
    --region us-east-1
  3. Fix Content-Types: 
    # JavaScript files need application/javascript
aws s3api copy-object \
    --bucket ffj-consulting-website \
    --copy-source ffj-consulting-website/assets/index-*.js \
    --key assets/index-*.js \
    --content-type application/javascript \
    --metadata-directive REPLACE \
    --profile my-sso
  4. Invalidate CloudFront Cache: 
    aws cloudfront create-invalidation \
    --distribution-id E3545N3N8YO2FZ \
    --paths "/*" \
    --profile my-sso

Content-Type Configuration

Important: JavaScript files must have application/javascript content-type for ES modules:

  • HTML: text/html
  • JavaScript: application/javascript
  • CSS: text/css
  • Images: Auto-detected (image/png, image/jpeg, etc.)

Cache Management

CloudFront Cache Invalidation

After deploying updates, invalidate the cache:

aws cloudfront create-invalidation \
    --distribution-id E3545N3N8YO2FZ \
    --paths "/*" \
    --profile my-sso

Cache Headers

  • HTML: Cache-Control: no-cache
  • JS/CSS: Cache-Control: public, max-age=31536000, immutable
  • Images: Cache-Control: public, max-age=31536000

Monitoring & Maintenance

CloudWatch Metrics

  • Monitor CloudFront requests
  • Track error rates
  • Monitor bandwidth usage
  • Set up alarms for issues

Cost Optimization

  • Use CloudFront caching to reduce S3 requests
  • Enable compression in CloudFront
  • Use appropriate price class
  • Monitor and optimize cache hit ratio

Troubleshooting

Common Issues

  • MIME Type Errors: Fix JavaScript content-type to application/javascript
  • 404 Errors: Configure CloudFront error pages for SPA routing
  • Cache Issues: Invalidate CloudFront cache after updates
  • SSL Errors: Verify certificate is validated and attached
  • DNS Issues: Check Route 53 records and propagation

  • Always use HTTPS (CloudFront with SSL)
  • Set correct content-types for all files
  • Invalidate cache after deployments
  • Monitor costs and usage
  • Keep deployment scripts updated
  • Document all configuration changes
