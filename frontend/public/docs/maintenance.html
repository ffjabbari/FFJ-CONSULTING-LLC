Maintenance

Overview

This guide covers ongoing maintenance tasks, updates, troubleshooting, and best practices for keeping the FFJ Consulting LLC website running smoothly.

Regular Maintenance Tasks

  • Update dependencies (monthly)
  • Review and update content (as needed)
  • Monitor performance and errors
  • Backup important data
  • Security updates

Content Updates

Adding New Articles

  1. Create markdown file in frontend/src/content/
  2. Add entry to frontend/src/data/articles.json
  3. Reference the markdown file in the contentFile field
  4. Rebuild and deploy: 
    cd frontend
npm run build
python3 deploy.py

Updating Existing Content

  • Edit markdown files directly in src/content/
  • Update JSON metadata in src/data/
  • Rebuild and redeploy

Updating Resume

  1. Edit frontend/src/content/resume.md
  2. Or use the update script: python3 update-resume-docx.py
  3. Rebuild and deploy

Dependency Updates

Frontend Dependencies

Check for updates:

cd frontend
npm outdated

Update packages:

npm update

Update major versions (carefully):

npm install react@latest react-dom@latest

Backend Dependencies

Update .NET packages:

cd backend/FFJConsulting.API
dotnet list package --outdated
dotnet add package [PackageName]

Update Process

  1. Review changelogs for breaking changes
  2. Update dependencies in development
  3. Test thoroughly
  4. Fix any breaking changes
  5. Deploy to production

Performance Monitoring

Key Metrics

  • Page Load Time: Target < 3 seconds
  • Time to First Byte (TTFB): Target < 600ms
  • Largest Contentful Paint (LCP): Target < 2.5 seconds
  • Cumulative Layout Shift (CLS): Target < 0.1

Monitoring Tools

  • Google PageSpeed Insights: Performance analysis
  • CloudWatch: AWS service metrics
  • Browser DevTools: Network and performance tabs
  • Lighthouse: Automated performance audits

Error Monitoring

Common Issues

  • 404 Errors: Check routing configuration
  • MIME Type Errors: Verify content-types in S3
  • CORS Errors: Check backend CORS configuration
  • Build Failures: Check dependencies and Node.js version

Logging

  • Check browser console for frontend errors
  • Review CloudWatch logs for AWS issues
  • Monitor backend logs for API errors

Security Maintenance

Regular Security Tasks

  • Update dependencies to patch vulnerabilities
  • Review AWS IAM permissions
  • Check SSL certificate expiration
  • Review access logs for suspicious activity
  • Keep AWS CLI and tools updated

SSL Certificate

ACM certificates auto-renew, but verify:

aws acm list-certificates --profile my-sso --region us-east-1

Backup & Recovery

What to Backup

  • Source code (Git repository)
  • Content files (markdown, JSON)
  • Database (if using backend database)
  • Configuration files

Backup Strategy

  • Code: Git repository (GitHub)
  • Content: Version controlled in Git
  • S3: Enable versioning if needed
  • Database: Regular automated backups

Troubleshooting Guide

Website Not Loading

  1. Check CloudFront distribution status
  2. Verify S3 bucket is accessible
  3. Check DNS records in Route 53
  4. Verify SSL certificate is valid
  5. Check browser console for errors

Content Not Updating

  1. Invalidate CloudFront cache
  2. Clear browser cache
  3. Verify files uploaded to S3
  4. Check file content-types

Build Failures

  • Check Node.js version (should be v18+)
  • Delete node_modules and reinstall
  • Check for dependency conflicts
  • Review error messages carefully

Optimization

Performance Optimization

  • Optimize images (compress, use WebP)
  • Enable CloudFront compression
  • Minify CSS and JavaScript
  • Use lazy loading for images
  • Implement code splitting

Cost Optimization

  • Monitor AWS usage and costs
  • Optimize CloudFront cache settings
  • Use appropriate S3 storage class
  • Review and optimize CloudFront price class

  • Update dependencies

Best Practices

  • Test changes in development before production
  • Keep documentation updated
  • Use version control for all changes
  • Document any manual configuration changes
  • Monitor and respond to issues promptly
  • Keep deployment scripts updated
