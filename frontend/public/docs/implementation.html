Implementation

Code Structure

The FFJ Consulting LLC website is implemented using modern web development practices with a clear separation of concerns and modular architecture.

Project Organization

FFJ-CONSULTING-LLC/
├── frontend/
│   ├── src/
│   │   ├── components/       # React components
│   │   ├── data/            # JSON data files
│   │   ├── content/          # Markdown content
│   │   ├── config.js         # Configuration
│   │   └── App.jsx          # Main app
│   ├── public/               # Static assets
│   └── dist/                 # Build output
├── backend/
│   └── FFJConsulting.API/   # C# API
└── Docs/                     # Documentation

Frontend Implementation

React Components

The frontend is built with React 18 using functional components and hooks:

Navigation Component

  • File: src/components/Navigation.jsx
  • Features: Responsive menu, smooth scrolling, active state management
  • State: Uses useState for menu toggle, useLocation for route tracking

Landing Page Component

  • File: src/components/LandingPage.jsx
  • Features: GPU/TPU animation, smooth transitions
  • Animation: CSS animations with React state for interactivity

Hero Component

  • File: src/components/Hero.jsx
  • Purpose: Main hero section with call-to-action
  • Styling: Gradient background, centered content

Content Components

  • AITechnologies.jsx: Displays AI technology cards from JSON data
  • Articles.jsx: Lists articles with markdown content rendering
  • Tools.jsx: Shows development tools and resources
  • Resume.jsx: Renders resume from markdown file
  • ArticleDetail.jsx: Individual article page with full content

Data Management

JSON Data Files

  • articles.json: Article metadata (title, description, category, contentFile)
  • technologies.json: AI technology information
  • tools.json: Development tools and resources

Markdown Content

  • Location: src/content/ and public/content/
  • Format: Standard markdown with frontmatter support
  • Rendering: React Markdown library for parsing and display

Routing Implementation

React Router is used for client-side routing:

import { BrowserRouter, Routes, Route } from 'react-router-dom'

<Routes>
  <Route path="/" element={<LandingPage />} />
  <Route path="/home" element={<LandingPage />} />
  <Route path="/resume" element={<Resume />} />
  <Route path="/article/:articleId" element={<ArticleDetail />} />
</Routes>

State Management

  • Local State: useState for component-level state
  • Navigation State: useLocation and useNavigate from React Router
  • Data Loading: useEffect for fetching and loading content

Backend Implementation

ASP.NET Core API

The backend is implemented using ASP.NET Core 8.0:

Program.cs Structure

  • Services: Database context, CORS, Swagger configuration
  • Middleware: Routing, authentication, error handling
  • Endpoints: API controllers and routing

Controllers

  • BackendInfoController: Returns backend information (name, version)
  • Future Controllers: Articles, Technologies, Tools (as needed)

Database Context

  • File: Data/ApplicationDbContext.cs
  • Database: SQLite3 (development), can be switched to PostgreSQL
  • ORM: Entity Framework Core

Build Process

Frontend Build

Vite is used as the build tool:

# Development
npm run dev

# Production Build
npm run build

Build Output

  • Location: frontend/dist/
  • Contents: Optimized HTML, CSS, JavaScript bundles
  • Optimization: Code splitting, minification, tree shaking

Development Workflow

Local Development

  1. Start Backend: cd backend/FFJConsulting.API && dotnet run
  2. Start Frontend: cd frontend && npm run dev
  3. Or Use Script: python3 start.py (starts both)

Content Updates

  • Edit JSON files in src/data/ for metadata
  • Edit markdown files in src/content/ for content
  • Changes appear on refresh (hot reload in dev mode)

Testing

  • Frontend: http://localhost:3000
  • Backend API: http://localhost:5000
  • Swagger UI: http://localhost:5000/swagger

Key Implementation Details

Content Loading

Content is loaded dynamically:

  • JSON Data: Imported directly in components
  • Markdown: Loaded using dynamic imports or fetch
  • Images: Served from public/images/

Error Handling

  • 404 Pages: React Router handles unknown routes
  • API Errors: Try-catch blocks with user-friendly messages
  • Loading States: Loading indicators during data fetch

Performance Optimizations

  • Code Splitting: Route-based code splitting
  • Lazy Loading: Components loaded on demand
  • Image Optimization: Proper sizing and formats
  • Bundle Size: Tree shaking removes unused code

Configuration

Frontend Config

src/config.js contains:

  • SITE_URL: Production website URL
  • GITHUB_REPO: GitHub repository link
  • API_URL: Backend API endpoint

Backend Config

appsettings.json contains:

  • Backend:Name: Backend identifier (C#, Java, Python)
  • ConnectionStrings: Database connection
  • CORS: Allowed origins
